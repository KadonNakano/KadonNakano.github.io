When building a website, it is easy for the code to turn into a messy pile of buttons, pages, forms, and data all mixed together. Design patterns help prevent that. A design pattern is basically a common way to solve a common coding problem. It is like following a recipe instead of guessing every step. The pattern does not write the project for you, but it gives you a cleaner way to organize the pieces so the project is easier to build, fix, and explain later.

For our final project, Student Event Hub, the main idea was to make a web app where students could find student life events, create new ones, see all events, and save events to their own calendar page. Since our project was based on Bowfolios, we used a similar kind of structure where the site is split into pages, forms, reusable parts, and data collections. That means we were using a design pattern without always thinking about it as some fancy term. The app has one part that handles what the user sees, another part that deals with event information, and another part that connects the two when someone creates, edits, filters, or saves an event.

For our final project, we used Bowfolios as the framework for the entire project. Instead of starting with a blank page, we used its structure as a guide for how our own web app should be organized. Bowfolios already had a layout for pages, forms, navigation, user accounts, and data, so we were able to build Student Event Hub on top of that same kind of setup. We changed the purpose of the project from portfolios to student life events, but the basic design pattern stayed similar. This helped us focus more on adding our own event features, like creating events, viewing all events, and saving events to a personal event list, instead of having to figure out the whole structure from scratch.

In the end, design patterns helped keep Student Event Hub organized and easier to build. Bowfolios gave us the framework, and we used that structure to turn a portfolio app into a student event app. Instead of starting from nothing, we followed a proven setup. That helped us focus on all of the features that set Student Event Hub apart from bowfolios.

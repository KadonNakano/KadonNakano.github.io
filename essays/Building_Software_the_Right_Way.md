## More Than Just Code

<img src="/img/studyimg.png" alt="Studying software engineering" width="250" align="left" style="margin-right: 24px; margin-bottom: 18px;">

Before this class, I thought software engineering was mostly about writing code and getting an app to work. Since the class used tools for making web applications, it would be easy to think the whole class was just about building websites. But I learned that web development was really just the setting. The bigger lesson was learning how real software projects are built with other people. Two ideas that stood out to me were **open source software development** and **coding standards**, because they go hand in hand.

**Open source software development** means working on software where the code is shared openly so other people can view it, use it, report problems, and sometimes help improve it. This is bigger than just web apps because open source projects can be anything: apps, games, tools, libraries, operating systems, or programming languages. But when a lot of people can work on the same project, the code needs to be clear and organized. That is where **coding standards** come in. Coding standards are rules or guidelines for how code should be written, named, formatted, and organized. They help make sure the code is easier for everyone to read and understand.

These two ideas connect because open source software depends on people being able to understand each other’s work. If everyone writes code in a totally different style, the project can quickly become confusing. Coding standards make it easier for new people to join, fix bugs, and add features without having to guess what is going on. Overall, this class taught me that software engineering is not just about making something that works on my own computer. It is about writing code that other people can read, use, improve, and trust.
